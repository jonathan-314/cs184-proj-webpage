Simulation of Solids and Fluids

Team Members: Andrew Huang, Jonathan Guo, Wentinn Liao

In this project we will simulate fluids and solids as they interact with each other by modeling interactive forces with various equations. Doing so will allow us to compute updates to the scene each time-step, and render the progression accordingly. Solid objects can be modeled by a combination of exerted forces and torque, while fluids can be modeled as a finite set of small spherical masses that are governed by interactive forces.

Problem Description

Fluid movement is a rather complicated task, as there are not only collisions with solids but also intermolecular forces. Our goal is to simulate a fluid as it interacts with itself and other solid objects in the scene. We will model fluid as a finite set of point masses that are governed by intermolecular forces, gravity, and collisions with other point-masses and solid objects. To make the computation of pairwise interactions between elements more efficient, we can use spatial hashing to reduce the quadratic time complexity.

Goals and Deliverables

In this project we will simulate fluid-solid interactions by modeling fluid as a finite set of point-masses. This will involve modeling interactive forces between water molecules, object collision, and computation of a surface mesh for rendering.

Since we are doing a simulation, we will use videos instead of images. The hope is that rendering each frame will not take too much time.

We plan to deliver a working simulation of solids, including momentum shifts and collisions. We hope to deliver a working simulation of fluids and solids interacting with each other. We think it may be easier to code liquid simulation, but that it likely takes significantly longer to render. We hope to include various types of fluids in our simulation - for example, fluids with different viscosities and densities, and more complex processes such as laminar and turbulent flow. We also, given enough time, aim to explore collisions with more complex solids. We do anticipate that there will be a decent rendering time, especially when we add fluids, so we are unsure whether we will realistically create a full simulation for liquids.

Our main two methods of performance measurement are speed and realisticness. It is rather easy to test the rate at which a particular frame is rendered but slightly harder to determine how realistic it is. One possible solution is to do a comparison of multiple collision functions.

In our analysis, we plan to discuss and compare different collision and intermolecular force functions, not only referring to research but also our own tests.
