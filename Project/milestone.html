Current Progress

Currently, we’ve refactored all code mentioning the cloth from Project 4 to fluids instead as that is what we are focused on. As for simulations, we’ve accounted for all intermolecular forces in addition to plane and spherical collision. Our constructor employs a static array, partitioning the space into segments/cubes and only considering forces between molecules in adjacent or the same cubes in order to speed up runtime. In computation of forces, all computations of terms in the Navier-Stokes equation are done in batch: that is, we compute the values for particles within the same cell simultaneously, which allows us to save many computations. Our numerical integration process uses adaptive time-stepping. At the beginning of the simulation frame, the maximum velocity of all particles is computed, and a timestep is taken to be inversely proportional to the maximum velocity. In order to prevent either very low or very high timesteps, we clamp with a minimum granularity and maximum step size. We then regulate collisions by iterating through all solid objects stored with the Fluid class. After velocities and positions have been updated, we reallocate all particles to their new respective cells to be in the correct neighborhoods for the next simulation step.

Preliminary Results

Our simulation code runs pretty fast which is a good sign. We are currently able to accurately simulate position and velocity updates to point masses, using forward adaptive integration without velocities blowing up. However, we have not yet completed the rendering portion yet so we are currently unable to tell if our model is realistic or not.

Reflection and Updates

Due to the complexity of the physics governing large solids, we have abstained from modeling both dynamic fluids and solids and instead will model the fluid interactions with static solid environments. We can do this by expanding to finite plane collisions, which allows us to simulate more interesting environments.

During the upcoming week, we plan to finish configuring our code and adding the mesh of fluid. Afterwards, we may play with different viscosities and setups, such as different configurations of collision objects.
